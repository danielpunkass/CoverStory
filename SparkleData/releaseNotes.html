CoverStory Release Notes

Please consider joining our mailing list at http://groups.google.com/group/coverstory-discuss.

Issues can be filed on our issues list at http://code.google.com/p/coverstory/issues/list.

Version 4.0

  • Sparkle Integration
    We now should be autoupdating with Sparkle.
  • Export To HTML
    You can now export your data out to HTML, using the "Export To HTML" command from the File menu. The HTML is quite customisable by supplying your own coverstory.css file, and coverstory.js files.
  • AppleScript
    We have added comprehensive AppleScript support that should allow you to easily extract any coverage information you want out of CoverStory. Using AppleScript it is easy to create an Xcode build step that will take your gcov files and generate HTML to post to a website, or to keep track of coverage from build to build. Take a look at our AppleScript dictionary for complete details.
  • Removed Code Complexity
    We removed the code complexity feature as it seemed to be muddying up the interface, and wasn't being used very much.
  • Added Toolbar
    We added a pretty toolbar. What more can we say?
  • Warns About Non-feasible Markers on Executed Lines
    You now get warnings if you execute lines that you have marked as non-feasible. If you marked them as non-feasible, we assume you didn't expect them to be executed.
  • Fixed Up Document vs. Application Preferences
    Several preferences that were previously application wide are now document settings as they should be.
  • Many bug fixes
    Many other little bug fixes and features have been added.
